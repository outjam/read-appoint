# Требуется обновление
Фича предназначена для предотвращения ситуаций, когда часть пользователей остаётся на устаревших версиях приложения, что может приводить к ошибкам, некорректной работе или невозможности использования новых возможностей. Механизм основан на отображении специального уведомления при запуске приложения или в момент получения информации об устаревшей версии.

### Рекомендация — `Soft Update`
Приложение уведомляет пользователя о доступной новой версии, но не блокирует работу.

### Обязательное обновление — `Force Update`
Пользователь не может продолжить работу, пока не установит последнюю доступную версию.

### Критичное обновление — `Critical Update`
Пользователь не может продолжить работу, пока не установит другую доступную версию.

**Далее действует одно из правил:**

### Рекомендация — `Soft Update`
Если текущая версия < рекомендуемой, но ≥ минимально допустимой:

- Показывается уведолмение о доступном обновлении.
- Пользователь может:
  - Перейти к обновлению
  - Продолжить работу (кнопка «Позже» или закрытие окна)

### Обязательное обновление — `Force Update`
Если текущая версия < минимально поддерживаемой:

- Показывается блокирующий экран.
- Пользователь **не может продолжить работу** без обновления.
- Доступна только кнопка «Обновить».

## Админ-панель
Позволить администраторам управлять политикой обновлений, задавая минимально допустимую версию приложения, а также конфигурацию поведения для устаревших версий.

**Управление версиями**
Администратор может настроить параметры для каждой платформы (iOS/Android):

- **Минимально поддерживаемая версия** — при её снижении приложение должно заблокировать доступ для версий ниже указанной (режим Force Update).
- **Рекомендуемая версия** — приложение показывает уведомление, но продолжает работать (режим Soft Update).
